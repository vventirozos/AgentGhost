tools / image_gen.py

Stable Diffusion XL image generation routed to an external GPU node.

Tool: image_generation

Auto-registered when context.llm_client.image_gen_clients exists.

  • Heals missing prompt by checking related kwargs (image, description, subject, text).
  • Clamps steps to [4, 8] (Lightning models hate longer schedules).
  • Calls llm_client.generate_image({"prompt", "steps"}).
  • Decodes base64 PNG, writes to sandbox as gen_UUID.png.
  • Returns a markdown embed with a short alt text (![generated image](/api/download/gen_UUID.png)) — the giant raw prompt is deliberately kept out of the alt text.
  • Instructs the model to follow the embed with one or two short sentences describing what it generated, in its own words, so the user always gets a human-readable caption instead of a bare link.

Backend topology and SDXL model config: image-gen server.

2026-07-14 audit fixes

  • The SUCCESS message now states the actual output size. The requested dimensions are snapped to the image node's bucket ladder (SD1.5 VRAM envelope), but that adjustment was only logged for the operator — the model told the user it generated the requested size, or re-called the tool trying to "fix" a size that was deliberately snapped. The message now says Rendered at WxH (snapped from …).
  • The sandbox dir is created before the write. A fresh project scope may not have created the dir yet; without the mkdir a FileNotFoundError discarded an image the GPU node had already spent ~30s producing. Covered by tests/test_tools_batch_audit_fixes.py.
