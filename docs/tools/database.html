tools / database.py

PostgreSQL administration with parameterised queries, schema introspection, plan analysis, and pooled connections.

Tool: postgres_admin

ActionBehaviour
queryRun arbitrary SQL with a 15 s statement timeout (configurable). Output clamped to 300 rows. Returns tabulated results from a RealDictCursor.
schemaDump information_schema.columns for public tables. Filterable by table_name via parameterised query.
explain_analyzePrepend EXPLAIN (ANALYZE, BUFFERS, FORMAT TEXT) if not already present; runs with the same timeout.
activitySelects from pg_stat_activity; bounded to 50 longest-running rows.

Security

  • All schema introspection uses parameterised queries (cur.execute(sql, (table_name,))).
  • SQL is passed through extract_code_from_markdown() first so fenced SQL blocks are stripped cleanly.
  • Connection pool with autocommit; broken connections are closed and discarded; the next call opens a fresh one.

2026-07-14 audit fixes

  • Host-restriction bypass via libpq URI query params (MED, closed). When a default DSN is configured, an LLM-supplied connection string may only target the SAME database. The check was built from urlparse netloc alone — but libpq honours connection parameters in the URI QUERY STRING, and ?hostaddr=10.0.0.99 is the actual TCP target (the netloc host becomes mere TLS SNI), while ?host=/?port=/?dbname= likewise override it. So …/prod?hostaddr=10.0.0.99 passed the netloc check and connected to 10.0.0.99 (verified with psycopg2.extensions.parse_dsn). The comparison key is now built by parsing the URI directly with urllib (urlparse + parse_qs), resolving the query-string overrides — hostaddr preferred over host (libpq's endpoint precedence), last-duplicate-wins — so all four override shapes are caught. Parsed with urllib rather than psycopg2.parse_dsn on purpose: the guard stays deterministic even where a partial/mocked psycopg2 makes parse_dsn unavailable (a parse_dsn returning a non-dict would have silently reopened the bypass). A non-numeric port in the supplied URI now returns a formatted error instead of an uncaught ValueError.
  • confirm="false" no longer authorizes DROP/TRUNCATE (MED, closed). Tool args arrive as strings, and bool("false")/bool("0")/bool("no") are all True — so an explicit confirm="false" was passed to the validator as confirmed. Coercion is now affirmative-token only (true/1/yes/y).
  • schema output is now row-capped (1000 columns) with an explicit truncation note — a wide DB previously flooded the model context with every column of every public table, unbounded (the inverse of the query/activity caps).
  • Session statement_timeout no longer leaks across pooled calls — it is SET at the top of every action, so a prior query with a short timeout can't cause spurious cancellations on a later schema/activity reusing the cached autocommit connection.
  • The SQL validator still fails OPEN on its own crash (a validator bug must not brick a mostly-read tool) but now logs at WARNING, not debug — the operator sees when the destructive-statement guard is disabled. Covered by tests/test_tools_batch_audit_fixes.py.

DSN

Default DSN comes from --default-db / GHOST_DEFAULT_DB. The tool can be invoked with an alternative DSN per-call.
