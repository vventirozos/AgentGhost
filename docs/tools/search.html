tools / search.py

Anonymous web search via DuckDuckGo + Tor, with reformulated fallbacks and multi-source synthesis.

Reformulation strategy

  1. Drop years / version numbers / numbers from the query (broaden).
  2. Convert to a question form ("how to X").
  3. Append tutorial / guide / example.

Output cleaning

_clean_for_cpp preserves valid JSON but strips control characters and structural braces from non-JSON text. This prevents the C++ JSON parser inside llama.cpp from crashing on stray {…} in scraped page bodies.

Stylometric egress scrubbing

Tor anonymises the network layer; it does nothing about the content layer. The agent's prose — politeness habits, first-person framing, punctuation idiosyncrasies, verbosity — is a stable author fingerprint that would otherwise egress verbatim to every search engine. utils/stylometry.py normalises the outbound query into a neutral keyword form before it leaves the box, under --anonymous (content anonymisation travels with network anonymisation).

  • scrub_query — deterministic, always-safe lexical pass used by web_search: peels stacked request/politeness frames ("please can you find me…", "I'm looking for…"), lowercases, drops trailing sentence punctuation, removes politeness tokens — while preserving every content keyword and in-word technical symbol (c#, c++, .net survive). Idempotent.
  • neutralize_query — stronger tier used by deep_research (already LLM-heavy): re-authors the query into a neutral keyword search via Ghost's own upstream, falling back to scrub_query on any failure. No third party.

Covered by tests/test_stylometry.py.

Per-identity Tor circuit isolation

Under --anonymous, each search also rides a distinct Tor circuit: helpers.socks_url_with_identity injects a per-query SOCKS username:password into the proxy URL, and Tor's IsolateSOCKSAuth (on by default) maps distinct credentials to distinct circuits — so a colluding set of exit nodes / sites can't trivially link a sequence of different searches into one session. No NEWNYM thrash, no control port needed. The tag is a hash of the (scrubbed) query, so identical repeated queries reuse a circuit while different queries separate. Covered by tests/test_quick_wins.py.
