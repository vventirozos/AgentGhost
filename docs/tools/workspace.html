tools / workspace.py

Read-only view of the user's workspace state — file watchlist, scheduled-task outcomes, research artifacts, command outcomes, running narrative.

Why this tool exists

The workspace continuity module's wake-up prefix is already spliced into every system prompt, so casual workspace questions ("what's been going on?") are grounded by default. This tool is the explicit handle for deterministic snapshots and ad-hoc queries the wake-up prefix doesn't cover.

It is the world-model counterpart to introspect (which reads the agent's selfhood). The writable counterpart is workspace_track.

Parameters

FieldTypeNotes
actionenumOne of summary, stats, files, changes, tasks, research, commands, narrative, recent. Defaults to summary.
limitintFor tasks / research / commands / recent: how many entries to return. Defaults to 10; clamped to [1, 50].

Failure semantics

  • Workspace disabled (--no-workspace-model / --no-memory): returns a single human-readable "unavailable" sentence.
  • Unknown action: returns "SYSTEM ERROR: 'action' must be one of …".
  • Internal exception: caught at the tool boundary and returned as a short failure string. Introspection never breaks a user turn.

Tests

tests/test_workspace_tool.py covers every action including the empty / no-tracked-files / disabled / None / invalid-action paths, plus registry wiring.
