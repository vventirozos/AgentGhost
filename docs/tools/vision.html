tools / vision.py

Multi-modal analysis via the visual LLM pool. PDF rasterisation to JPEG; URL or file inputs.

Tool: vision_analysis

Auto-registered when the LLM client supports vision.

ActionDefault prompt
graph_analysisAnalyse this graph/chart. Extract key data points, trends, legends, and conclusions.
describe_pictureDescribe this image in high detail. Mention objects, text, people, colors, and layout.
extract_text_pictureExtract all text from this image exactly as written (OCR).
extract_text_pdfExtract all text and describe any diagrams from these document pages exactly as written.

PDF handling

PyMuPDF rasterises the first 10 pages at 2× zoom to JPEG; otherwise the host's vision LLM would balloon context. A single image file is base64-encoded directly. 2026-07-14: the 10-page cap is no longer silent — when the PDF has more pages, the result carries an explicit only the first 10 were analyzed note pointing at read_chunked/knowledge-base ingestion for the rest. The fitz branch is also gated on the file actually being a PDF: previously action='extract_text_pdf' forced it for ANY target, so calling it on an image replaced the already-extracted image data with a doomed PDF parse — for a non-PDF target that action is now just a prompt choice (OCR over the image).

Local-file typing (2026-07-14)

Local files are typed from CONTENT first — magic-byte sniffing (_sniff_image_mime: PNG/JPEG/GIF/BMP/TIFF/WEBP) — with the original target name's extension as fallback. A file that matches neither is refused with a pointer to file_system(operation='read'). Previously an unknown type was silently guessed image/jpeg, so a .txt/.bin was shipped to the vision model to hallucinate over. Sniffing also fixes extensionless screenshots. The prompt parameter heals the aliases the model reaches for (question/query/text/instruction).

Execution

Calls llm_client.chat_completion({"messages": [...]}, use_vision=True). Returns "VISION ANALYSIS RESULT:\n{response}".

Safety

  • 50 MB cap on both local files and URL downloads (streamed with a byte cap, so a lying Content-Length can't OOM the host).
  • Path-traversal guard via _get_safe_path, with the read-only sandbox-root fallback for project-scoped sessions.
  • SSRF-on-redirect closed (2026-07-14). URL fetches used follow_redirects=True while validating only the ORIGINAL url — a public page 302-redirecting to 127.0.0.1/169.254.169.254/a LAN host bypassed the guard (the exact hole closed in tool_download_file on 2026-07-07; vision never got that fix). Auto-redirect is now OFF; every hop's Location is re-validated via the shared _download_redirect_target, bounded at 5 hops. Covered by tests/test_vision_hardening.py.
