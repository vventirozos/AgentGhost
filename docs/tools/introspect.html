tools / introspect.py

Read-only handle the agent reaches for when asked to describe itself, recall what it has been working on, or surface specific past turns.

Why this tool exists

The selfhood wake-up prefix is already spliced into every system prompt (see Selfhood), so a casual "tell me about yourself" is already grounded in the running diary. introspect is the explicit handle for cases where the agent wants a deterministic snapshot rather than relying on whatever the prefix happened to surface — answering specific recall questions, producing a structured self-report, or letting evaluation scripts pull a known shape.

Counterpart to self_state: introspect reads, self_state writes. The two are deliberately split so a single tool description does not conflate "introspect myself" with "author my forward-looking continuity slot".

Parameters

FieldTypeNotes
actionenumOne of summary, stats, narrative, recent, recall. Defaults to summary.
querystringRequired for action="recall". The phrase to search the autobiographical log for.

Failure semantics

  • Selfhood disabled (--no-self-model / --no-memory): the tool returns a single human-readable "unavailable" sentence. It does not raise.
  • Unknown action: returns "SYSTEM ERROR: 'action' must be one of …".
  • Missing query on recall: returns "SYSTEM ERROR: 'query' is required for recall.".
  • Internal exception in the SelfModel read path: caught at the tool boundary, logged at WARNING, returned as a short failure string. Introspection must never break a user turn.

Wiring

Declared in tools/registry.py:

  • TOOL_DEFINITIONS entry adjacent to self_state, with the read-only intent emphasised in the description so the model picks the right tool when asked introspective questions.
  • get_available_tools(context) dispatch lambda binds context.self_model into tool_introspect — the same pattern self_state uses.

Tests

tests/test_selfhood_introspect_tool.py covers:

  • Default action renders the summary block (stats, recent experiences, first-person framing).
  • Explicit action="summary" matches the default exactly.
  • stats reports correct counts, current mood, and at least one topic cluster.
  • narrative reports "No narrative" when empty; returns persisted diary text when present.
  • recent returns the last-N experiences, gracefully handles the empty-log case, and clamps absurd limits.
  • recall requires a query, surfaces relevant past turns, and rejects unrelated ones via IDF scoring; the no-match case is explained quietly.
  • Disabled / None SelfModel returns the "unavailable" message rather than crashing.
  • Invalid action returns the system-error string.
  • An exception in the underlying recall path is swallowed — the tool never re-raises.
  • Registry wiring: introspect appears in TOOL_DEFINITIONS with the expected enum, and the dispatch lambda binds context.self_model.
