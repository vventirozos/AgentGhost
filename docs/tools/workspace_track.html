tools / workspace_track.py

Write path into the workspace continuity module — author the watchlist of files to track, free-form workspace notes, and manual research dedup markers.

Why this tool exists

The workspace continuity module passively records research artifacts, scheduled-task outcomes, and tracked-file diffs. But it doesn't decide on its own which files matter or what is worth marking as already-seen. workspace_track is how the agent (or, by proxy, the user) authors that.

Counterpart to the read-only workspace tool. Same shape as the self_state + introspect pair on the selfhood side.

Failure semantics

  • Workspace disabled: returns a clear "unavailable" sentence. Nothing is recorded.
  • Missing required parameter: returns "SYSTEM ERROR: …" identifying the missing field.
  • Idempotency: tracking the same path twice keeps the first label; mark_seen for an already-known URL reports "already on the seen list".
  • Internal exception: caught at the tool boundary; the user-facing turn is never broken.
