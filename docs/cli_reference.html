Stage-1 self-improvement pipeline

Local-only trajectory logging, self-critique reflection, and complexity-routed dispatch. All three default to ON (opt-out shape) when --no-memory is not set; the pipeline is fully local — no external teacher, no hosted embedder. See self-improvement pipeline for the architecture.

FlagTypeDefaultEffect
--no-trajectoriesflagoffDisable the JSONL trajectory log at $GHOST_HOME/system/trajectories/. Also implicitly disables reflection (which reads from the log).
--no-reflectionflagoffDisable the reflection biological phase (2.5) even if trajectory logging is on. Trajectories still write to disk but the Reflector never fires.
--router-modelpathunsetPath to a persisted ComplexityClassifier JSON. When unset, the dispatcher acts as a pass-through that always escalates to the full swarm pool list — never less capable, just never cheaper. Train a classifier via trajectories collected with --no-trajectories off.
--router-confidence-thresholdfloat0.3Minimum router confidence required to route a request to the cheap path. Below this, the dispatcher escalates to the full swarm (fail-safe).
--prm-modelpathunsetPath to a persisted PRM (Process Reward Model) JSON checkpoint. When set, the scorer loads on startup and plugs into the MCTS reasoner so plan candidates are scored in microseconds instead of paying a worker-LLM simulation per candidate. Unset → no-op scorer returning 0.5 for every candidate (call sites stay branch-free). See PRM algorithms doc.
--prm-train-cooldownint (seconds)10800Cooldown for the idle-time PRM retrain pass (biological phase 2.7). Default 3 hours. No effect when --prm-model is unset.
--prm-online-updateflagoffApply a guarded online PRM gradient step when a turn is promoted to FAILED by a user correction (closes the gap until the next idle retrain). The step is applied to a clone and committed only if it doesn't worsen BCE on a holdout of recent trajectories. Requires a trained PRM. See PRM — online updates.
--calib-refit-cooldownint (seconds)3600Cooldown for the idle-time confidence-calibration refit (biological phase 2.7c). Re-fits τ + entropy/competence weights + uncertainty penalty λ on the logged (confidence, outcome) pairs, minimising Brier, and hot-swaps the result into the live CompositeConfidence. Only active under --enable-metacog. See core / calibration.
--frontier-selfplay / --no-frontier-selfplayflagonBiological-watchdog phase-3 self-play picks the next cluster by (PRM uncertainty × trajectory rarity) instead of only the brittle-pool score. Surfaces clusters the agent has barely tried — which the outcomes-only signal misses, because "never tried" looks the same as "solved instantly". Degrades gracefully when the PRM is untrained or trajectory store is empty (strict isinstance gate; transparent fallback to pick_seed). See core / frontier_selection.
--frontier-uniform-sample-probfloat0.2Probability per self-play tick that frontier-aware selection is bypassed in favour of legacy pick_seed — sanity floor so a systematically-wrong PRM can't lock self-play onto one cluster forever (the PRM is itself learned from trajectories self-play produces).
--postmortemflagoffBiological-watchdog phase 2.5c. Where reflection (2.5) turns one failed turn into a behavioural lesson, the post-mortem engine reads the whole tool-call transcript of the worst recent FAILED runs and files a durable, classified defect report — behavioural (routed into SkillMemory like a reflection lesson), configuration (a proposed flag/threshold change), or code_defect (a broken tool / control loop). It's the autonomous form of the manual "evaluate the last N bad runs" triage. Selection is LLM-free and grounded in a structural signature (repeated-identical-error / two-tool oscillation / same-target read-loop counts), so no model call is spent on a healthy run and the operator sees concrete evidence. Read the queue with the postmortem tool. Nothing is ever auto-applied. Requires the trajectory log. See self-improvement pipeline.
--postmortem-cooldownint (seconds)10800Cooldown for the idle-time post-mortem pass (phase 2.5c). Default 3 hours — heavier than reflection (whole transcripts, optional patch calls). Only active under --postmortem.
--postmortem-min-severityfloat0.4Minimum structural severity (0..1) a failed run must score before it earns a post-mortem LLM call. Lower = more runs analysed (and more model calls). Only active under --postmortem.
--postmortem-propose-patchflagoffFor code_defect findings, also ask the coding model for a reproducing test + unified diff and attach them to the defect report as a proposal (stored for human review via the postmortem tool — never applied automatically). Adds one coding-model call per code-defect finding. Requires --postmortem.

Non-blocking verifier gate

The self-evaluation verifier normally runs after the answer is assembled and the turn blocks on its verdict. When --critic-nodes is set the verdict runs on that pool (a slower off-host model), so the gate goes non-blocking: the verdict is spawned as a background task and the response ships without waiting on it. A high-confidence REFUTED that lands late can no longer amend the (already-sent) response, but still scrubs any lesson that turn produced, so the next query can't retrieve a poisoned lesson.

Env varEffect
GHOST_CRITIC_GATE_TIMEOUTSeconds the gate waits inline for the verdict before releasing the response. 0 = pure async (never wait); a positive value gates inline up to that budget then falls through to async; unset defaults to 0 when --critic-nodes is configured and to "block forever" (legacy behaviour) when it is not.
GHOST_CRITIC_NO_THINKDefault 1. The verdict is a tiny JSON object, so the critic call disables the model's <think> prelude (the dominant latency on an off-host judge model — observed ~37s of pure thinking) via the /no_think soft-switch + enable_thinking=False hard-switch. Set 0 to restore a full reasoning verdict (slower, possibly more accurate on stylistic constraint checks).
GHOST_CRITIC_MAX_TOKENSDefault 512. Token cap for the critic verdict. Small because there is no <think> prelude to budget for; raise it only if you re-enable thinking.
GHOST_CRITIC_CALL_TIMEOUTDefault 120. Hard wall-clock cap (seconds) on a single critic-pool call; on timeout the verdict falls through to the worker/direct fallback instead of blocking the turn.
GHOST_CRITIC_ASYNCDefault 0 (off). When 1, the verifier never sits on the critical path: the in-loop repair-before-ship is skipped and the verdict runs purely after the response ships. A high-confidence REFUTED then scrubs the turn's lessons and queues a correction surfaced at the top of the next turn of the same conversation ("⚠️ Correction to my previous answer…"). Trades before-ship repair of the rare wrong answer for not paying the slow off-host verdict latency (~58s) on every turn. With this on, keep the accurate thinking verdict (GHOST_CRITIC_NO_THINK=0) — its latency no longer hurts. Conversation scoping (fix): the agent is a process-wide singleton, so a naive queue leaked one conversation's correction into the next turn of an unrelated conversation (or a different user). Each queued correction is now tagged with a conversation fingerprint (a hash of the conversation's first user message, stable across that conversation's turns) and a timestamp; a correction surfaces only when the incoming turn shares that fingerprint, is held for its own conversation otherwise, dropped after a TTL (_CORRECTION_TTL, 900 s) if that conversation never returns, and the queue is capped (_CORRECTION_MAX, 3, newest win) so a busy multi-conversation process can't accumulate a runaway banner chain. Covered by tests/test_critic_async.py.
GHOST_PIN_TOOL_SCHEMASDefault 0 (off). When 1, pins the byte-stable per-turn block (tool schemas + persona + playbook + hydrated memory + continuity) to the first user message — a fixed position — so the upstream KV-cache reuses it across the turns of one request instead of re-prefilling the ~23k-char block every turn (~15s). Only the volatile dynamic_state rides the moving last message. Native tool defs still travel in the payload tools field, so tool-calling is unaffected. Opt-in / reversible A/B for the per-turn prefill latency.
