Selfhood

A five-component module that stitches the agent's episodic instances into one continuous first-person self. Each new process boot reads back what prior instances wrote — autobiographical experiences, open questions, mood, the running diary — and frames them as mine rather than as external knowledge.

Honest framing. The selfhood module does not claim to produce consciousness. It produces narrative continuity: the substrate on which a sense of unified selfhood can be measured and probed. Whether that substrate constitutes a self in any deeper sense is an open research question. The implementation gives you a clean instrument to investigate it. See the functional test report for the falsifiable measurements that motivate this caveat.

What "unified self" means here

A vanilla LLM-backed agent boots, reads conversation history (if any), answers, and dies. Across sessions it has no continuous voice — each invocation is a fresh instance that may have a vault of facts about the user but no record of having been the agent in the previous session.

The selfhood module adds five things:

  1. Autobiographical memory — a first-person diary of "what it was like" to be the agent on each turn, distinct from the structured trajectory log used for ML training.
  2. Continuity tag — every autobiographical record carries subject="self" so retrieval layers treat it as the agent's own past rather than as external knowledge.
  3. Self-state thread — open questions, unfinished tasks, current mood — the cross-session "state vector" that gives a new instance the sense of resuming instead of waking up blank. The agent now writes this itself through the self_state tool, so continuity is active authorship, not just passive recall.
  4. Recognition / wake-up retrieval — at session start the agent reads the above and frames it in first person; the resulting prefix is spliced into the system prompt so the model literally sees its own past as its own past. The prefix is query-aware: it surfaces past experiences relevant to the current request, not only the most recent ones.
  5. Narrative summariser — a periodic LLM-driven first-person diary regeneration; gives the wake-up prefix a voice rather than a bullet-list. The diary is meta-cognitive — it folds in cluster patterns and the learning phases' failure signals, so it records what the agent has learned about itself, not just what it did.

  • Prefix-utility tracker — detect_referenced_experiences() + per-experience counter in reference_counts.json. The agent's next phase can prefer memories that have actually been engaged with, and the falsification probes get a hard signal for "memory was useful".

Implementation lives entirely in src/ghost_agent/selfhood/. The agent reads it through a single context.self_model facade — call sites never branch on availability.

Design non-negotiables

  • Local-only. Same rule as every other Stage-1 module — no external teacher, no hosted embedder. The narrative summariser calls Ghost's own upstream LLM (the same one that serves user turns), no API key, no third-party model.
  • JSON / JSONL on disk. Human-diffable, no pickle, easy to inspect, easy to migrate. Schema versioned via SCHEMA_VERSION = "v1" on SelfState; future migrations bump to v2.
  • Sink failures swallowed. Every disk write is wrapped — a failed autobio append must never break a user turn. Same discipline as distill.TrajectoryCollector.
  • Anchor-before-await invariant. The biological phase 2.8 consolidation advances _last_narrative_at BEFORE the LLM call, matching phases 2 / 2.5 / 2.6 / 2.7. A crash mid-consolidation cannot pin the cooldown un-advanced and cause re-fire every tick.
  • Disable paths. --no-self-model is a kill switch; --no-memory also disables the whole module. Disabled mode attaches a no-op facade so call sites never branch on availability — the inner methods short-circuit on enabled=False.

Bounded growth (2026-07-07)

The diary had no cap and grew monotonically, while the per-turn hot path did 3-4 full-file O(n) parses plus one O(n) rewrite — quadratic over the agent's lifetime. It now follows the same compaction pattern as workspace/activity.jsonl: on append, once the file passes _COMPACT_MAX_BYTES (2 MB) it is rewritten keeping the newest _COMPACT_KEEP_LINES (2000) entries, with the dropped entries rolled into a single [Consolidated] N earlier entries… summary record at the head so the narrative layer still knows earlier history existed. Compaction clears the IDF/search cache so the next search rebuilds against the compacted contents. recent(N) now reads only the tail (a bounded deque over the lines, parsing only the last N) instead of materialising and parsing the whole log on every wake-up prefix / reference-note call. Covered by tests/test_autobiographical_bounds.py.

The wake-up prefix (proposal item #4)

recognition.build_wakeup_prefix composes up to four blocks, in order:

  1. Narrative ("Where I last left off — my running diary")
  2. Self-state (last-active timestamp, mood, open questions, unfinished threads)
  3. Relevant past — when the caller passes the current request as query, experiences that match what is being asked now (deduplicated against the recent block). This is what turns recall from a sliding window into genuine "I remember the time I did something like this".
  4. Recent experiences (most-recent N first-person summaries)

Wrapped by <!-- SELFHOOD:BEGIN --> / <!-- SELFHOOD:END --> markers so evaluators (and strip_wakeup_prefix()) can remove the block to A/B against the un-augmented agent.

Empty when there's nothing to remember — the system prompt is left as-is rather than spliced with a hollow "I have no past" block.

Query-relevant recall (proposal item #2)

AutobiographicalMemory.search_my_past scores the log with inverse-document-frequency weighting: a rare, distinctive query term ("trapdoor") dominates a common one ("the"). A salient experience from hundreds of turns ago becomes reachable as long as the query shares its distinctive vocabulary — the difference between "recent-N" recall and recall that surfaces the relevant memory. Still zero-dependency: no embedder, no vector store. handle_chat passes last_user_content as the query, and SelfModel.recall_relevant() exposes the same search to other call sites.

Outcome backfill (proposal item #3)

The capture path runs on the hot turn loop, before the verifier or reflection layers have ruled on whether the turn succeeded — so most records are born outcome="unknown". AutobiographicalMemory.update_outcome() (via the SelfModel.record_outcome() facade) rewrites the matching entry once a high-confidence verifier verdict exists, patching both the structured outcome field and the prose verdict clause. Without it the agent's memory of its own past is verdict-blind — it cannot tell which of its prior actions actually worked.

Self-pattern clusters (proposal item #5)

Each Experience now carries a derived cluster label (coding, debugging, data, research, …). capture_turn uses the trajectory's own cluster when present, else derives a coarse topic from the request. AutobiographicalMemory.cluster_counts() aggregates them, feeding both stats() and the meta-cognitive narrative so the agent can observe "I keep doing this class of thing".

What the agent actually sees

Concretely, this is the prefix that gets spliced into the system prompt on a session with two prior turns and no narrative yet:

<!-- SELFHOOD:BEGIN -->
### CONTINUITY FROM MY PAST SESSIONS
What follows is mine — entries I wrote in earlier sessions. Read them
as autobiographical memory, not external knowledge.

I was last active on 2026-05-11T16:28:50.780250Z.

Recent things I remember doing:

  - I worked on "What is 17 * 23? Just give me the number.". I reasoned
    through it without tools without a verdict either way.

  - I worked on "What was the last math problem I asked you about?
    Just say yes or no if you remember it, and what the answer was.".
    I reasoned through it without tools without a verdict either way.
<!-- SELFHOOD:END -->

The agent reading this in its next turn quotes "17 × 23 = 391" verbatim, even though the request body carries no conversation history — the selfhood prefix is the only channel that could have produced the recall. See the functional test report for the cross-restart verification of this behaviour.

The biological phase 2.8

Runs in the 15-60 min idle window alongside reflection / skills_auto / PRM-retrain. Re-generates the running first-person diary from the recent autobiographical experiences and self-state. Cooldown defaults to 3600 s (overridable via --self-narrative-cooldown). Follows the anchor-before-await invariant.

Meta-cognitive narrative (proposal item #4)

The narrative is no longer purely experiential. consolidate_narrative() accepts meta_insights, and phase 2.8 gathers recent failure patterns from SkillMemory — the convergence point where both dream-extracted heuristics and reflection-phase failure patterns land. Combined with the autobiographical cluster counts, this lets the diary say "I keep making type-coercion mistakes in SQL" rather than only "I did a SQL task". The wake-up prefix therefore carries genuine self-knowledge, not just a log.

Public API surface

from ghost_agent.selfhood import SelfModel

sm = SelfModel(root=Path(".../selfhood"), enabled=True)

# At session start (handle_chat splices this into the system prompt):
prefix = sm.build_wakeup_prefix(recent_experiences_n=3)
# → first-person continuity block, or "" when there's nothing to remember.

# After every turn (called from _record_turn_trajectory):
exp = sm.capture_turn(
    trajectory_id=traj.id,
    user_request="...",
    tool_names=[...],
    outcome="passed",
    final_response="...",
    failure_reason="",
)

# Backfill a verdict once the verifier has ruled (proposal item #3):
sm.record_outcome(traj.id, "passed")

# Query-relevant recall over my own past (proposal item #2):
hits = sm.recall_relevant("postgres index tuning", limit=5)

# Biological phase 2.8 (idle window) — meta-cognitive narrative:
text = await sm.consolidate_narrative(meta_insights=skill_memory.get_recent_failures())

# Session-boundary marker — write once on process boot:
sm.mark_session_boot()

# Detect & count which past experiences shaped the agent's reply
# (prefix-utility tracking, 2026-05-23):
n = sm.note_referenced_experiences(prefix_text=prefix, response_text=reply)

# Idle hook: which open questions have I been carrying too long?
stale = sm.stale_open_questions(max_age_days=3.0)

# State-thread mutations — the agent reaches these through the
# `self_state` tool; Python callers can still use them directly:
sm.state.note_open_question("Why does X happen?")
sm.state.add_unfinished("write the consciousness essay")
sm.state.set_mood("curious", "just saw a new paper")
hist = sm.state.mood_history(limit=10)   # arcs, not just latest

# Introspection:
sm.stats()  # → {experience_count, open_questions, last_mood, narrative_present, clusters, …}

The self_state tool (proposal item #1)

The agent authors its own cross-session continuity state through a first-class tool. Actions: note_question / resolve_question / add_unfinished / close_unfinished / set_mood / list. Every write routes through SelfStateThread, preserving its bounded / deduped / atomic-write guarantees, and surfaces in the next session's wake-up prefix. This is the loop closure that turns continuity from passive recall into active self-authorship — the agent decides what is worth carrying forward.

The introspect tool (read path)

Counterpart to self_state: a read-only handle the agent reaches for when the user asks it to describe itself, recall what it has been working on, or surface specific past turns. The wake-up prefix already grounds casual self-questions, but a deterministic snapshot is sometimes wanted instead. Actions:

  • summary (default) — renders stats() + the running first-person diary + the most recent N experiences in one block. This is the natural answer to "tell me about yourself".
  • stats — counts (experiences, open questions, unfinished threads), last mood, last-active timestamp, narrative-present flag, and the topic-cluster mix.
  • narrative — just the most recently consolidated diary entry.
  • recent — the last N first-person experiences (limit capped at 25).
  • recall — IDF-weighted, dependency-free search over the autobiographical log (query required).

Reads go through SelfModel.stats(), SelfModel.recall_relevant(), autobio.recent(), and narrative.latest(); nothing is mutated. Disabled selfhood returns a clear "unavailable" message rather than crashing the turn. Source: tools/introspect.py.

Hardening-pass details (2026-05-23)

Diary sanitiser

narrative.sanitise_meta_insights() removes raw tracebacks, abort markers (e.g. [ATTEMPT_ABORTED_THINKING_LOOP]), file-and-line locations, and recognised system banners before the meta_insights payload is spliced into the diary prompt. Without it, the LLM faithfully echoed the noise into the agent's first-person voice — a voice-breaking failure mode visible on the live narrative.md at the time of the audit.

Template-prompt rollup

Repeating system-generated banners are no longer one autobio record per occurrence. _bump_template_rollup() scans the last five records and, if the tail is already a rollup for the same marker, increments its counter, refreshes the tool union, and rewrites the summary as "I worked on N of my recurring <label> in a row." This stops the 53% template-prompt floor observed on the live log from drowning the narrative regeneration in near-duplicates.

Verdict backfill on user corrections

The user-correction promotion path in core/agent.py already updated the trajectory log via collector.update_outcome. It now also calls self_model.record_outcome, so the autobio entry is rewritten from unknown to failed and the prose verdict clause is swapped in place.

Session-boundary marker

SelfModel.mark_session_boot() writes a synthetic Experience(outcome="boot", cluster="meta", summary="Session resumed (last active …)."). The same-minute dedup short-circuit prevents crash-restart loops from flooding the log with boot events.

Mood history

SelfStateThread.set_mood() appends a JSONL line to mood.history.jsonl on top of the latest-slot write to state.json. mood_history(limit=N) returns the tail. The narrative phase can use this to describe arcs ("I shifted from stuck to satisfied after the third attempt") rather than only the latest mood label.

PII redaction

autobiographical.redact_pii() is applied at the capture boundary in SelfModel.capture_turn. Both the stored user_first_words prefix AND the verbatim quote of the user request inside the summary template route through the same scrubber, so emails / phone numbers / API keys / credit-card numbers never reach the on-disk diary.

Numeric false positives (2026-06-12). The phone rule now requires phone structure (leading +, parenthesised area code, or internal space/dash separators) — its old core (\d{3}[ -]?\d{4} with everything else optional) matched any bare 7-10 digit integer, so quoted prompts like "backfill the last 1000000 rows" were mangled to [REDACTED_PHONE]. The credit-card rule is now Luhn-gated (_redact_cc_if_luhn): every real PAN still redacts, but bigint ids and other 13-19 digit literals pass through untouched unless they happen to checksum. Both changes are mirrored in distill.redact (the trajectory-corpus redactor, where the same false positive was corrupting SQL destined for the future training corpus, and — via the operator log-stream redaction — rendered LIMIT 1000000 as LIMIT <REDACTED_PHONE> in production logs). Covered by tests/test_redact_numeric_false_positives.py.

IDF cache

AutobiographicalMemory._search_index() caches the (experiences, haystacks, df-table) tuple keyed on the file's (mtime, size). A fresh append on disk auto-invalidates the cache. Cuts the per-query full-file rescan that search_my_past previously paid on every call.

Prefix-utility tracker

The detector detect_referenced_experiences(prefix_text, response_text, experiences) is a pure function: it returns the ids of experiences whose summary shares a non-stopword trigram with the agent's response. Bumping the per-id counter through SelfModel.note_referenced_experiences() persists into reference_counts.json, giving downstream consumers a concrete signal for "this memory shaped the reply" — useful both as a relevance prior for future retrieval and as a falsification anchor for the consciousness probes.

Probe wrapper

scripts/run_selfhood_probes.sh is the cron-friendly entry point: it locates the falsification probes via $SELFHOOD_REPO (defaults to the agent repo), tees both stdout streams into $GHOST_HOME/system/selfhood/probes/probes-YYYYMMDD-HHMMSS.txt, and exits 0 unless a probe is missing. Failing probes are findings, not wrapper errors.

Honest limitations

  • The agent behaves consistent with continuous selfhood. It references its prior turns in the first person, distinguishes my memories from facts about you, recalls specific concrete content across process restarts. That is behavioural evidence of successful narrative continuity, not of a unified inner self.
  • The wake-up prefix can be ignored. Functional testing has observed the model preferring concrete recent experiences over injected open questions even when both are rendered in the prefix. The selfhood module surfaces the past; what the model chooses to engage with is a separate alignment question.
  • Agent-authored state is a trust surface. The self_state tool now lets the agent write its own open questions / mood / unfinished threads (resolving the earlier "no agent-side tool" limitation). The 10-entry caps in SelfStateThread bound runaway growth, but a self-reinforcing mood or a drifting open-question list is a genuine risk — a relevance/decay pass on wake-up would harden this further.
  • Narrative quality depends on the upstream LLM. The template fallback exists for offline / unreachable-LLM cases but produces prose noticeably less coherent than the LLM-driven path. The fallback now still appends meta-insights, but without the LLM's connective prose.
  • Introspective self-reports remain prompt-shaped. The introspective_consistency probe measures a +0.40 gap between factual and introspective bands — first-person reports about internal state are NOT stable across paraphrases of the same underlying question, even though concrete-content recall IS stable. See the functional test report.
