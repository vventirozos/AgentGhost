Workspace Continuity

The world-model counterpart to selfhood. Selfhood tracks "who I am"; workspace continuity tracks "what I'm looking at."

Why this exists

Selfhood was added to give a fresh session of the agent a sense of resuming rather than waking blank — autobiographical diary, open questions, mood, recognition prefix. That closed the asymmetry on the self side. But the world side was still snapshot-only: every session the agent saw an unannotated current file listing, no record of what its scheduled tasks did overnight, no memory of which URLs it had already pulled, no diff against the user's last engagement. The agent's self-model had become richer than its world-model.

Workspace continuity is the symmetric module that fixes this. Same shape as selfhood:

  • An activity log (append-only JSONL) mirrors the autobiographical log.
  • A state thread (single JSON file) mirrors the self-state thread.
  • A recognition layer renders a wake-up prefix spliced into every system prompt, right after the selfhood prefix.
  • A narrative summariser regenerates a "running summary of the workspace" in the idle phase 2.9, immediately after the selfhood narrative phase 2.8.
  • A WorkspaceModel facade on context.workspace_model is the only object the rest of the agent talks to.

What it tracks

KindWhat gets recordedHow
file_changedOne entry per tracked file whose stat changed since the last scan (newly tracked / modified / appeared / deleted).On every build_wakeup_prefix() call: scan_tracked() stats every TrackedFile against its cached FileSnapshot. The diff is what the wake-up prefix surfaces.
task_outcomeOne entry per scheduled-task firing: job_id, name, passed/failed, duration, summary, error.The APScheduler callback _run_proactive_task in main.py sinks success/failure into WorkspaceModel.record_task_outcome in a try/except so a failing sink can't kill the scheduler.
researchOne entry per unique URL pulled by deep_research; second-and-later occurrences silently dedup against the seen-URL set.tool_deep_research receives workspace_model via the registry lambda and calls record_research_artifact on every discovered URL.
commandSignificant execute-tool outcomes (long-running, failed, mutating). Caller decides what's significant.WorkspaceModel.record_command_outcome. Wiring into tool_execute is the next phase.
noteFree-form workspace events authored by the agent via the workspace_track tool.WorkspaceModel.note.

Module layout

FileResponsibility
src/ghost_agent/workspace/schema.pyDataclasses: FileSnapshot, TrackedFile, TaskOutcome, ResearchArtifact, CommandOutcome, WorkspaceEvent, WorkspaceState.
src/ghost_agent/workspace/activity.pyAppend-only JSONL log of WorkspaceEvent. Same discipline as selfhood.autobiographical: sink failures swallowed, mid-file corruption tolerated. Self-compacting (2026-07-03): once the file passes _COMPACT_MAX_BYTES (2 MB) on append, it is rewritten in place to the newest _COMPACT_KEEP_LINES (2000) events — the log is read in full on the per-turn prefix path, so it must not grow unbounded. iter_events() snapshots the raw lines under the lock and parses OUTSIDE it (holding a non-reentrant lock across yield risked a deadlock on append-during-iteration); recent() uses a bounded deque so it is O(limit) memory, not O(file).
src/ghost_agent/workspace/state.pyWorkspaceStateThread — persistent JSON state (tracked files, seen-URL set, last + prior session timestamps). Atomic temp+rename flush. Bounded slots: MAX_TRACKED_FILES=200, MAX_SEEN_URLS=5000. Tracked paths are expanduser+abspath-normalised at track time (a raw ~/x is never stat-able; a relative path drifts with the CWD). touch_session() rolls the previous timestamp into prior_session_at before stamping the boot, so the cross-session last touched line survives. A corrupt state.json is moved aside as a state.json.corrupt-<ts> sidecar instead of being silently overwritten by the next flush, and one malformed tracked_files entry is skipped rather than discarding the whole state. File-watcher lives here: scan_tracked() diffs current stat against cached snapshot.
src/ghost_agent/workspace/recognition.pybuild_workspace_prefix() — pure read function that composes activity + state + narrative + file-diffs (+ broken-file warnings) into the prefix the agent splices into the system prompt. Markers <!-- WORKSPACE:BEGIN --> / END for evaluator strip.
src/ghost_agent/workspace/reactions.py 2Acheck_changed_python_files() — first consumer of the file_changed diffs. For each changed .py it cheaply ast.parses the file (decoding as utf-8-sig with errors="replace" so a BOM'd but valid file isn't falsely flagged and an invalid-byte file the interpreter would reject isn't silently passed); any that no longer parse are returned as {path, error} and rendered as a loud Files I touched that currently DO NOT PARSE block in the wake-up prefix, so the agent fixes a file before relying on it mid-task. Pure, synchronous, never raises; deleted/non-Python/oversized files are skipped. (Runtime ImportErrors need execution — out of scope for this static check; a nearest-test re-run behind a flag is the planned follow-up.)
src/ghost_agent/workspace/narrative.pyWorkspaceNarrative — periodic LLM-rewritten "running summary of the workspace". Template fallback when no critique function is supplied, same pattern as selfhood.narrative. Also exposes render_changelog() 2B — rolls the activity log into a dated, project-scoped markdown changelog; WorkspaceModel.write_changelog() persists it as CHANGELOG.<project>.md during the idle Phase 2.9 consolidation (alongside the prose narrative).
src/ghost_agent/workspace/model.pyWorkspaceModel — single facade. Capture hooks (record_task_outcome, record_research_artifact, record_command_outcome, note), file watcher (track_file, untrack_file, scan_tracked), prefix builder (build_wakeup_prefix), idle consolidator (consolidate_narrative → also writes the changelog), introspection (stats). record_navigation() 2C counts same-URL visits this session and, on the 3rd, returns a "cache it / switch strategy" nudge that the browser tool appends to its result (and records as a workspace note).

Wiring into the agent

  1. Lifespan construction (main.py): context.workspace_model = WorkspaceModel(root=$GHOST_HOME/system/workspace/, …) — same place where context.self_model is built. mark_session_boot() stamps the session.
  2. Prompt assembly (core/agent.py): right after the selfhood prefix splices in, workspace_model.build_wakeup_prefix() is invoked under a strict isinstance gate (defends against MagicMock test contexts) and the resulting block is prepended to base_prompt. Empty when disabled or no state.
  3. Scheduled-task ledger (main.py::_run_proactive_task): around the existing handle_chat dispatch we record outcome="passed" on success and outcome="failed" with the exception class+message on failure. Both branches are wrapped in try/except so a sink failure can't take down a cron job.
  4. Research dedup (tools/search.py::tool_deep_research): after URL discovery and before page fetching, we record every URL into the workspace. The seen_urls set is what makes the next research call dedup.
  5. Idle narrative (core/agent.py::_biological_tick phase 2.9): a new phase added immediately after the selfhood narrative phase 2.8. Same idle window (15-60 min) and cooldown discipline (_WORKSPACE_NARRATIVE_COOLDOWN=3600, overridable via --workspace-narrative-cooldown).

Agent-callable tools

  • workspace — read-only. Actions: summary (default), stats, files, changes, tasks, research, commands, narrative, recent. Answers "what changed since yesterday?", "what did my scheduled task do?", "did I already pull this URL?".
  • workspace_track — write. Actions: track, untrack, note, mark_seen. The agent authors the watchlist and records free-form workspace observations.

Relation to selfhood

The two modules are designed to coexist. Selfhood answers "tell me about yourself" with the agent's autobiographical past and ongoing internal state. Workspace continuity answers "what's happening in my workspace?" with the user's external state. Both wake-up prefixes are spliced into the same system prompt, with selfhood first (who I am) and workspace second (what I'm looking at). The clean separation is intentional: an evaluator can strip one block without disturbing the other.
