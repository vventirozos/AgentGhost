Mandatory Tor — runtime fail-closed egress

Makes the README's "fails closed if Tor is unreachable" promise real. Source: utils/egress_guard.py. ON by default; opt out via --no-mandatory-tor or GHOST_MANDATORY_TOR=0.

The gap it closes

Historically the promise was aspirational: every outbound helper degraded to a direct cleartext connection when the proxy was absent or down — core.llm.get_proxy returns None for a missing proxy, tools.search adds proxy= only if tor_proxy is set, and the SSRF/identity-rotation paths all guard on if tor_proxy. A blank TOR_PROXY silently de-anonymised every request. The only true fail-closed mechanism (eval.network_guard) ran only during offline evals.

The policy

When --mandatory-tor is set, a process-wide guard monkeypatches the outbound socket primitives (connect / connect_ex / sendto / sendmsg) and raises MandatoryTorError on any connection to a globally-routable (public) address. Allowed: loopback, RFC1918 private / LAN, link-local, multicast/reserved, and the configured SOCKS endpoint. Multicast is checked explicitly before the global-routability test: CPython reports 224.0.0.0/4 / ff00::/8 as is_global=True, which would otherwise wrongly fail-close mDNS/SSDP LAN discovery (fixed 2026-07-03).

Why that is exactly right and not over-broad:

  • Anonymised traffic egresses through the Tor SOCKS proxy at 127.0.0.1:9050 — the socket layer sees a loopback connect, which is allowed. With socks5h, even DNS is resolved by Tor's exit, so no public connect appears here.
  • Local infrastructure — the llama.cpp upstream, ChromaDB, a LAN Postgres, LAN swarm / image-gen nodes (e.g. 192.168.0.20, or a Tailscale/CGNAT GPU at 100.83.184.117) — is not globally routable, so it is allowed (such traffic does not deanonymise via a Tor exit, and a Tor exit cannot route a tailnet address anyway).
  • The ONLY thing blocked is a direct connection to a public IP — which by construction is a connection that bypassed Tor. That is precisely the leak the README claims is impossible.

Egress decision: one predicate, two enforcers

"Local-or-LAN vs. public" is decided by a single rule — not ip.is_global — applied in two places that must agree:

  • egress_guard.is_allowed_host — the fail-closed guard that blocks direct public connects.
  • core.llm.compute_tor_proxy — chooses, per outbound LLM/compute node, whether to route via Tor (public) or connect directly (local-or-LAN).

is_global is False for loopback, RFC1918, link-local, IPv6 ULA, and CGNAT / Tailscale (100.64.0.0/10). An earlier version of compute_tor_proxy used the narrower is_private or is_loopback, under which a Tailscale node reads as non-private — so an image-gen GPU at 100.x.x.x was forced through a Tor exit that cannot route a tailnet address, and every generation failed with ConnectError: All connection attempts failed. Aligning both call sites on is_global fixes that and guarantees the guard never blocks a node the client was told to reach directly.

Boot-time liveness gate

Before any outbound-capable component is wired, tor_liveness_ok() probes the SOCKS port. If Tor is unreachable and --mandatory-tor is set, the agent refuses to start (raises during lifespan) rather than booting cleartext — "a silently-cleartext agent is worse than a stalled one." Emits a Tor Fail-Closed log line (🛡️ active / ❌ abort).

Embedding model: forced offline

sentence-transformers / huggingface_hub fire a cleartext HTTPS model-resolution / version check at load time even for a cached model — a public connect the guard (correctly) blocks, which would otherwise stall the embedder at boot. Because Ghost's embedder is local-only by design, --mandatory-tor also sets HF_HUB_OFFLINE / TRANSFORMERS_OFFLINE (in _env.py, before the heavy imports) so the embedder loads from cache with zero network. The model must therefore be pre-cached — which it is after one normal (non-fail-closed) run. A cold install that enables fail-closed first should pre-download the embedder (or route HF through the SOCKS proxy and set HF_HUB_OFFLINE=0 to override). This also closes the small pre-existing cleartext leak that check represented.

Notes

  • Default OFF — existing deployments are never perturbed. Enabling it is the switch that turns the documented guarantee into an enforced one.
  • Inbound (uvicorn accept()) is unaffected — the guard only patches outbound connect/send*.
  • Complements the existing per-tool socks5h:// routing and --anonymous search rather than replacing it; this is the backstop that catches any path that forgot to route.
