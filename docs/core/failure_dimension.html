core / failure_dimension.py + failure_distill.py

Harness-dimension failure attribution and the dream-side failure-cluster distillation pass (MemoHarness adaptation, 2026-07-19).

Why

Three multi-day debugging episodes in this project's history were rooted in the same mistake — a harness failure read as a model failure: the verifier "not in evidence" refutes (evidence-packer truncation, not the judge), the code-correction failures (fail-open SEARCH/REPLACE parser, not the LLM), and the introspect breakage (native-tools arg corruption). Every failure record now carries a dimension naming the harness layer it was attributed to, so the debugging prior is audit the harness before blaming the model — and so recurring failures can be grouped and distilled into preventive lessons.

Precedence: first-match-wins over the tables in the order output_processing → memory → context_assembly → orchestration → tool_interaction → generation_control → model. Harness dimensions are deliberately checked before model, and specific beats generic — a network timeout (ETIMEDOUT) lands on tool_interaction while an LLM timeout (ReadTimeout) lands on generation_control.

The tag is a prior, not a verdict. Regex over failure text sees where a failure surfaced, not necessarily where it originated. Individual tags can be wrong; the aggregate distribution (a month of tags showing one dimension spiking) is the reliable signal. The matched signal is debug-logged at each write for spot audits.

Where attribution happens

  • Lessons — inside the single write chokepoint SkillMemory.learn_lesson: real-mistake lessons are classified from trigger + anti_pattern; mistake-less rules get no dimension; an explicit dimension= kwarg wins. Stored on the JSON row and the vector-twin metadata.
  • Work logs — the finalize chain captures failure heads next to classify_tool_failure (context._turn_failure_texts, capped at 6/turn) and classifies them (plus the verifier reason on a verifier:failed outcome) into the work_log payload's failure_dimension. Only failed turns pay for classification.
  • Counterfactual regressions — deterministically memory: a previously-passing challenge failing after lesson hydration means a learned lesson degraded behaviour.

failure_distill.py — the dream-side pass

Runs at the top of every REM cycle (before the entropy/idempotency gates — see dream cycle). Gathers the last 14 days of failure records from playbook + work_logs + counterfactual results, groups by (dimension, cluster) (cluster via frontier.classify_cluster), and for each group of ≥3 cases asks the worker model to distill ONE cross-case pattern lesson:

  • Write-back goes through learn_lesson with source="distilled", confidence=0.6, domains=[cluster, dimension] and the case handles as source_refs — so the existing hydration path retrieves it with zero new read-side plumbing.
  • The trigger is distilled(<dim>/<cluster>): <pattern>; on re-distillation the prior trigger is reused verbatim so dedup bumps frequency instead of adding rows (steady-state: one playbook slot per cluster).
  • An evidence fingerprint per cluster ($GHOST_HOME/system/failure_distill_state.json) skips clusters whose case set hasn't changed — same evidence would only re-mint the same lesson.
  • Records the heuristics left unknown are first re-classified by the worker (CLASSIFY_FAILURE route, ≤8/cycle) and the verdicts persisted onto the playbook.
  • Capped at GHOST_FAILURE_DISTILL_MAX (default 2) lessons per cycle; every path fails open to 0.

Knobs

Env varDefaultEffect
GHOST_FAILURE_DIM1Attribution at record time (lessons + work_logs). 0 kills.
GHOST_FAILURE_DISTILL1The dream-cycle distillation pass. 0 kills.
GHOST_FAILURE_ADJUDICATE1Worker-LLM re-classification of unknowns during the pass.
GHOST_FAILURE_DISTILL_MAX2Distilled pattern lessons per dream cycle.

All read per call (no restart needed) — the GHOST_VERIFY_TWO_STAGE idiom. Routing labels: CLASSIFY_FAILURE (adjudication, 8 tokens) and DISTILL_PATTERN (synthesis, 400 tokens, 60s) — both worker-only via LLMClient.route(), falling back to the heuristic / skipping on any failure.
