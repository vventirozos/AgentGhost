core / calibration.py — Calibration Spine

Closes the loop on confidence: pair each turn's composite confidence with the realized outcome, measure Brier / ECE, and re-fit τ + weights + λ to the data. Roadmap phase 2.5.

The problem it closes

The composite confidence was computed every turn but never checked against what actually happened. The threshold τ = 0.55 and the entropy/competence weights 0.5 / 0.5 were asserted constants — so "the agent is 80 % confident" had no demonstrated relationship to the turn succeeding 80 % of the time. The calibration spine turns that assertion into a measured, self-correcting loop.

The three jobs

  1. Measure. Every metacog turn produces a confidence reading paired with the realized outcome, appended to $GHOST_HOME/system/calibration/calibration.jsonl; from that we compute a rolling Brier score mean((C − outcome)²) and an Expected Calibration Error over a 10-bin reliability table.
    • Logprob-optional confidence. The composite is computed at turn finalization for every full-loop metacog turn — not gated behind the streaming is_final_generation+stream_response branch (which many turns never enter) nor behind a populated token-entropy window. The entropy term is used when the per-token window observed tokens and is neutral (0.5) otherwise, so on speculative-decoding / MTP / no-logprobs upstreams (where the per-token logprob stream is sparse or absent) competence + verbalised-uncertainty drive the score and a sample is still recorded. Verified live on the MTP llama-server: samples land with entropy_component=0.5, competence driving C.
    • Outcome signal. A turn is a negative (0.0) on a structural tool failure, a verifier REFUTED verdict (≥0.7), or a next-turn user-correction promotion (the strongest "confidently wrong" signal — recorded against the prior turn's stashed confidence); otherwise 1.0. Real negatives are what stop the fit from bailing single-class on free-form chat (which is otherwise almost all "clean").
  2. Self-tune. Once ≥ 40 samples (both outcome classes present) accumulate, an idle-time grid search re-fits the entropy/competence weights, the verbalised-uncertainty penalty λ, and the decision threshold τ to minimise Brier. The threshold is then chosen to maximise Youden's J on the "predict success when C ≥ τ" decision.
  3. Unify. The recorded sample carries the verbalised-uncertainty pressure alongside the objective entropy/competence components, so the formerly-disjoint "the agent said it was unsure" track and "the generation/domain was uncertain" track are fit together.

Design non-negotiables

  • Local-only, pure stdlib. math + json only — no numpy, no hosted scorer, no outbound traffic.
  • JSONL / JSON on disk. Append-only history, atomic param writes (.tmp + os.replace), schema-versioned (ghost.calibration.v1).
  • Fail-safe. A recording or fit failure is logged at debug and never breaks a turn; load_params returns None on any problem so a corrupt file degrades to the hardcoded defaults.
  • Bail-on-thin-data. Below 40 samples, or with only one outcome class present, fit returns None and writes no params — a confidently-miscalibrated threshold is worse than the neutral default (same posture as prm.trainer).

Observability

Refits emit metacog calib log lines (📐 icon) with the fitted threshold/weights/λ/Brier. tracker.stats() exposes {samples, brier, ece, fitted, threshold, w_entropy, lambda_uncertainty} for introspection.
