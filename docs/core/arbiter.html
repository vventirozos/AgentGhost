core / arbiter.py — DualSolverArbiter

Confidence-gated, two-shot consensus check. Runs two scripted-temperature candidates in parallel and routes the answer based on semantic divergence.

DISABLED on the request path since the 2026-06 cognitive-layer redesign. Gating constant: _METACOG_ARBITER_ENABLED = False (core/agent.py:106). A paired ablation found it net-negative — it sampled two paraphrased completions, computed their divergence, then threw both away and dispatched the ORIGINAL args, paying 2× inference for a number that gated a near-unreachable ask_user pause (dominant latency, zero answer changes). Re-enable criterion: only via its intended grounded replacement — verifier-judged best-of-N that SUBSTITUTES the winning answer, which landed 2026-07-07 as the async-critic bounded repair (verifier, IMPROVEMENTS.md #18). See PROJECT_JOURNAL.md §3 at the repo root. The code + tests below are preserved as a parked subsystem.

Decision tree

both candidates failed       → ask_user
one survived                 → execute survivor
both ok, converged (sim ≥ τ) → execute lower-T candidate
both ok, diverged:
    no validator             → validate (caller decides)
    validator passes one     → execute the passing one
    validator passes both    → ask_user (semantic disagreement)
    validator rejects both   → ask_user (with combined reason)

SemanticDivergence

Cosine similarity over an injected embedder. Production prefers the local vector-memory embedder (memory_system.embedding_fn, the all-MiniLM-L6-v2 SentenceTransformer) so the signal needs no model-server round-trip and works while the server runs in MTP/speculative mode — where --embedding is unavailable. It falls back to the remote LLMClient.get_embeddings endpoint, then (when neither is wired, e.g. tests with deterministic stubs) to token-overlap Jaccard — which under-clusters paraphrases, so the arbiter is slightly more cautious in that mode (the right failure direction).

Cost discipline

  • Two candidates only (not the five-sample default in distill/self_consistency.py).
  • Per-sample timeout (default 60 s, operator-tunable via --metacog-arbiter-timeout-s); a timeout is treated as a failure, NOT a halt. Each candidate is a full LLM completion routed over Tor, so the budget must clear real model latency (20–40 s). The previous 10 s default was shorter than a single completion, so both candidates timed out every turn and the arbiter degenerated into a constant ask_user; sized up so it actually arbitrates.
  • Empty prompt → skipped action; arbiter never doubles a no-op turn.

Convenience gate

confidence_gated(...) returns None when the composite confidence is at or above threshold, so call sites can hot-loop the arbiter call without rewiring the conditional themselves.
