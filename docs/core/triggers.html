core / triggers.py — first-class trigger taxonomy

Promotes the three trigger classes from the meta-cognition doc into events the planner can subscribe to, instead of scattering them across agent.py.

Event types

Bus

TriggerBus.publish fans out to kind-specific subscribers then wildcard "*" subscribers. Async coroutines are awaited; sync callables are called directly. Handler errors are isolated. The bus keeps a bounded history (default 64) for observability.

Helpers

SymbolRole
ToolRuntimeBudget(window=50, min_samples=10, multiplier=3.0)Rolling p95-runtime per tool. is_anomalous(tool, dur) answers the "execution anomaly" trigger. Cold-start safe.
RepetitionCounter(threshold=3)Sequence-aware streak counter — resets on a different key, not on a window slide. Tripped at threshold.
ReplanBridgeWildcard-subscribes the bus and forwards warning/critical events to ProjectPlan.request_revision (or plan.tree.request_revision). Audit log on .revisions.
loop_event / resource_event / anomaly_eventConvenience factories so producers don't manually populate every field.

Severity policy

  • info — observability only; ReplanBridge records it but does NOT call request_revision.
  • warning — triggers a replan if a current task id is known.
  • critical — same as warning, but caller policies may also halt new tool dispatch until the trigger clears.
