core / mcts.py — MCTSReasoner

Monte Carlo Tree Search for action selection with cached alternatives and backtracking.

Turn-start hint DISABLED on the request path since the 2026-06 cognitive-layer redesign. Gating constant: _MCTS_TURNSTART_ENABLED = False (core/agent.py:86). The MCTS "next action" was injected as a prompt block ending "a strong hint, not a mandate — deviate if needed", so the model regenerated freely and ignored it while the search cost ~4 LLM round-trips (the 390 s blow-ups). It is ungrounded: it scores the model's self-prediction of un-executed actions. Re-enable criterion: only once it has an execution-grounded value function (not self-prediction). See COGNITIVE_LAYER_REDESIGN.md at the repo root. The PRM that would feed it is likewise parked pending Track-B3 adjudication (IMPROVEMENTS.md #27b). Code + tests preserved as a parked subsystem.

Pipeline

  1. Expand — ask the LLM for N (default 3) distinct next-action candidates with description + tool + risk note.
  2. Simulate — for each candidate ask a worker LLM to predict outcome and rate progress / cost / risk in [0, 1].
  3. Score — combined score: 0.6 · progress + 0.15 · (1 - cost) + 0.25 · (1 - risk).
  4. Select — pick highest-scoring; push siblings onto a backtrack stack.
  5. Backtrack — when the chosen action fails, backtrack() pops the next-best candidate.

Concurrency

Fully async; asyncio.gather drives parallel simulation calls. Worker pool is preferred (cheaper, different perspective) when available.

Invocation in the turn loop

MCTSReasoner.select_best_action was constructed under --deep-reason but never actually called in the reasoning loop — dead wiring. It is now invoked.

  • For requests the complexity router classifies HARD (label "hard" or escalated), and when --deep-reason is on (so context.mcts_reasoner exists), handle_chat calls select_best_action and injects the top-ranked candidate as a "DEEP-REASON LOOKAHEAD" planning hint into the system prompt.
  • Bounded by a 75-second wall-clock timeout so a slow search cannot pin the turn.
  • Non-fatal — any failure is swallowed and the turn proceeds without the hint.

The router verdict is the gate, so the extra LLM round-trips land only where reasoning depth pays off.

Tests: covered by tests/test_mcts_lookahead.py.
