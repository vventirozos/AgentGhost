Project scoping of the wake-up prefix (2026-06-07)

The workspace model is global — one activity log + narrative shared across every project. That leaked a prior project's files into a new one: a fresh "create a Minecraft project" request inherited an earlier project's file://…/projects/991e52bce912/…/index.html research pulls in its wake-up prefix, then chased a file that didn't exist in the new project's (empty) sandbox — an infinite read loop.

Events are now scoped to the active project. WorkspaceEvent gains a project_id stamp (set at record time from WorkspaceModel.current_project_id, which the agent and the manage_projects _set_current path keep in sync). Stamping is race-safe (2026-07-09): every record_* site now reads a task-local ContextVar override first (workspace.model._EVENT_PROJECT_OVERRIDE) and falls back to the shared attribute only when unbound. The chat turn binds it via set_event_project() right where it syncs current_project_id; the idle autoadvance tick and the explicit-project manage_projects autoadvance batch wrap their tool runs in pinned_event_project(pid) — previously they pinned only context.current_project_id (the sandbox-scoping field) while events kept stamping from the shared model attribute, so an idle build's command outcomes were stamped with whatever project the last chat turn had open (deterministically wrong, plus a live race when a user turn overlapped an in-flight tick). Dream self-play's temp agent now detaches the shared model entirely (isolated_context.workspace_model = None): it runs under its own semaphore, so it both clobbered the global stamp pointer mid-turn and recorded synthetic outcomes into the real activity log. Covered by tests/test_workspace_event_stamping.py. At render time, build_wakeup_prefix(active_project_id) filters the recent-events list and tracked-file changes via filter_events_for_project, and drops the global narrative paragraph when it references a different project (_narrative_is_cross_project). An event's owning project is recovered from its explicit stamp or, for legacy events written before stamping, from a projects/<id>/ segment in its URL/path — so existing on-disk activity is scoped correctly with no migration. Project-agnostic events (web research, generic notes) are always kept; when no project is active, nothing is filtered. Covered by tests/test_workspace_project_scoping.py. Pairs with the agent-layer loop-breakers (repeated-identical-failure breaker) and the file_system loop-breaking "not found" message, which stop the resulting loop even if stale context slips through.
