core / entropy.py — token-level Shannon entropy

Pure-function entropy math + a streaming-friendly rolling tracker. The objective-calibration half of the composite confidence score.

Public API

SymbolPurpose
compute_token_entropy(top_logprobs) -> floatShannon entropy (nats) of one token's top-K logprobs. Re-normalises the truncation, ignores non-finite inputs.
normalise_entropy(h, k=5) -> floatMap raw entropy into [0, 1] by dividing by log(K). K=1 is coerced to 2 to avoid divide-by-zero.
EntropyTracker(window=32, top_k=5)Rolling-window accumulator. observe(top_lps) per token; reading() returns an EntropyReading (raw mean, normalised, n).
extract_top_logprobs(chunk) -> list[float] | NonePulls top-K logprobs out of an OpenAI-compatible streaming chunk. Recognises both logprobs.content[*].top_logprobs and llama.cpp's flat logprobs.top_logprobs. Lenient on shape; returns None on mismatch.

Invariants

  • Reading on an empty window: raw=0, norm=0, n=0. Callers can use n == 0 as the "no calibration yet" sentinel.
  • Uniform top-K → norm ≈ 1.0; peaked top-K → norm ≈ 0.0.
  • NaN / inf / non-numeric inputs are skipped, not propagated.
  • Logprobs are clamped to [-50, 0] before exp to prevent overflow.

Why no library?

SciPy's entropy would force a numpy dependency on every chunk parse — too expensive on edge. The pure-Python implementation is sub-microsecond per token and dependency-free.
