core / confidence.py — CompositeConfidence

Fuses token-level entropy (one-shot generation signal) and per-domain competence (long-term capability prior) into one tunable confidence score.

Formula

C = w_e · (1 - e) + w_c · p, with w_e + w_c = 1 enforced and both clamped to [0, 1]. When the competence cell has fewer than five observations, p is shrunk toward the neutral 0.5 by a coefficient n / (n + 5). A fresh agent never radiates overconfidence in a brand-new domain.

API

cc = CompositeConfidence(w_entropy=0.5, w_competence=0.5, threshold=0.55)
r  = cc.score(
    normalised_entropy=tracker.reading().norm,
    competence_p_success=profile.estimate("sql", "select"),
    n_observations=profile.observations("sql", "select"),
)
if r.below_threshold:
    # gate the dual-solver arbiter
    ...

Why additive, not geometric?

The two signals are asymmetric: entropy is per-generation; competence is historical. Weighted additive lets operators tune the balance per deployment — an edge build with a thin model raises w_c because token entropy is noisier there.

Fusing the verbalised-uncertainty track

Historically the objective score (entropy + competence) and the agent's own "I'm not sure" flags (core.uncertainty) were disjoint. score() now accepts uncertainty_pressure ∈ [0,1] (from UncertaintyTracker.pressure()) and applies a multiplicative penalty × (1 − λ·pressure). The default λ = 0 is a no-op, so this is byte-for-byte back-compatible — the calibration spine only raises λ if the agent's hedges actually predict failure on logged turns.

Self-calibration

apply_fitted(params) hot-swaps the threshold τ, weights, and λ from a calibration fit — at startup (loading a persisted fit) and from the idle refit phase (2.7c). A long-running agent's confidence becomes empirically calibrated against realized outcomes instead of the hardcoded τ = 0.55 / 0.5–0.5 defaults, with no restart.
