core / solution_novelty.py

Structural novelty of a candidate solution.py against prior winning solutions for the same cluster — closes the 2026-05-17 "no learning signal" gap.

Why

The pre-2026-05 self-play score used a tool-invocation count as a proxy for "how compressed is the plan". On deterministic templates the tool count is constant from one win to the next, so the compression delta pinned at +0.000 and the score collapsed to pure pass/fail. The frontier tracker saw a flat signal — see the post-mortem dated 2026-05-17 (no lessons learned after hundreds of cycles).

This module gives the scorer a real gradient: when a solver passes the same template a second time, did it find a structurally different solution, or did it just reproduce the prior winning shape verbatim? The former is real generalization; the latter is memorization and should not award learning credit.

What "canonical" means

The AST walker replaces variable names, attribute names, and literal values with placeholders before hashing. Two solutions that differ only in identifier names → same canonical hash → novelty 0.0.

# All three of these canonicalise to the same shape:
x = 1; print(x)
foo = 1; print(foo)
bar = 1; print(bar)

The Jaccard variant goes further: it computes the Jaccard similarity of bigrams in the canonical part stream and reports the distance. Captures local structural overlap rather than just whole-shape equality — a for-loop-with-accumulator differs from a comprehension-with-sum even though both produce the same answer.

Wiring

memory/frontier.FrontierTracker.record_run reads solution_source and stores both the source and the canonical hash in a small ring buffer (WINNING_SOLUTIONS_KEEP=5). The next cycle, core/dream.synthetic_self_play calls recent_winning_solutions(cluster) and passes the list into jaccard_novelty before recording the new run. The resulting novelty is plumbed into:

  • correctness_weighted_score (γ·novelty term),
  • the skill-write gate (a first-try pass with novelty ≥ 0.5 now writes a lesson, even when compression delta is flat),
  • the trajectory's extra.solution_novelty field — the reflector reads this to decide whether to meta-critique an otherwise boring pass.

Behaviour at the edges

  • Cold start (no prior wins for the cluster): novelty = 1.0.
  • Unparseable source (e.g. partial write, syntax error): novelty = 0.0 — broken solutions are not a learning signal even if they're textually unique.
  • Variable rename / whitespace change: novelty = 0.0 — same canonical shape.
  • Different structure (e.g. for-loop vs. comprehension): novelty > 0.3 typically; depends on local bigram overlap.
