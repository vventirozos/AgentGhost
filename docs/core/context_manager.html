core / context_manager.py

Progressive context compression. Avoids the binary "fits" / "emergency-prune" cliff by compressing in five graduated levels.

Wired into the hot path (2026-07-07, #27a). This module previously had ZERO production callers — the live path degraded in one cliff via core.agent's _prune_context (an LLM summarization pass). handle_chat now runs compress_if_needed(messages, max_level=3) immediately BEFORE _prune_context (via _get_context_manager(), lazily built with the real token estimator). The max_level=3 cap means only content compression runs (L1-L3) — every message is preserved, so tool-call/result pairing stays intact — and _prune_context remains the L4 emergency for the extreme case. Net effect: the expensive LLM summarization fires less often because the deterministic, LLM-free levels shed tokens first. Covered by tests/test_context_manager.py.

Levels

LevelUsed-token ratioEffectFull-fidelity turns
L0< 60 %No compression.all
L1< 75 %Summarize tool outputs (keep first & last 3 lines, error markers).last 6
L2< 85 %Collapse verbose assistant messages.last 4
L3< 95 %Compress user messages too.last 2
L4≥ 95 %Emergency prune — keep only system + last user + last tool output.1
