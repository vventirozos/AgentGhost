api / game_routes.py

Participant-mode games: the agent PLAYS, it does not delegate. A game-agnostic move endpoint over a registry of GameAdapters.

All endpoints require X-Ghost-Key. The router (prefix /api/game) is included before the catch-all proxy in app.py, like projects_routes.

Why this exists

"Play against the agent" must be literal: the client sends a position and the agent's LLM, at inference time, chooses the reply move. The endpoint owns the protocol — stateless, client-owns-the-state, code-enforced legality, bounded retries, and no engine fallback — and delegates each game's rules to an adapter. A silent engine fallback is exactly the "random AI" failure this endpoint was built to prevent (chess incident, 2026-07-02): if the model can't produce a legal move after _MAX_MOVE_ATTEMPTS (3), the endpoint returns 502 rather than picking a move for it.

Endpoints

Method · PathBehaviour
GET /api/game/gamesLists the registered games, e.g. {"games": ["chess", "tictactoe"], "default": "chess"}.
POST /api/game/moveApply the user's move (if any) to the state, then have the agent choose and apply its reply. Returns the updated state, the agent's move, an optional friendly comment, the move history, and game status. See the request/response shapes below.

Request

FieldMeaning
gameWhich game to play. Optional, default "chess". Unknown game → 400 with the available list.
stateSerialized state to move from (chess: a FEN; tic-tac-toe: 9 cells + side to move). Omit to start from the game's initial position.
fen[chess] Backward-compatible alias for state — existing chess clients that POST {fen, user_move?, history?} keep working unchanged.
user_moveThe user's move, applied before the agent replies. Omit the field entirely to ask the agent to move from the given state. An empty string is a client bug → 422 (a bare Enter forwarded as "" once made the agent play both sides, 2026-07-05).
historyMove list so far. Context for the agent, and the input some games replay to detect repetition endings (see chess below).

Response

A common envelope — {ok, game, user_move, move, comment, move_explanation, critique, state, history, attempts, …} — merged with the adapter's status (game_over, result, termination, turn, plus game-specific keys) and the move's extras. Chess additionally returns fen (alias of state), move_uci, check, and can_claim_draw. Status codes: 400 unknown game · 422 bad state / illegal or empty user_move · 501 a required library is missing (e.g. python-chess) · 502 LLM failure or no legal move after retries · 503 LLM client unavailable.

Coaching text — for learning clients

The endpoint surfaces the agent's own words about the position, so a client can build a learning experience (e.g. chess_client.py) rather than just a move relay. All three are game-agnostic and produced in the same single LLM call that chooses the move:

FieldMeaning
critiqueThe agent's read on the user's move — is it sound, what it threatens or overlooks, a stronger idea. Empty when the user didn't move this turn (the prompt only asks for it when there was a move to critique).
move_explanationThe reasoning behind the agent's own move — the plan, threat, or defence it serves.
commentA short friendly one-liner. Kept for backward compatibility; older clients that only know comment keep working, and a model that emits only a COMMENT: line still populates it.

Under the hood the adapter's prompt asks the model for MOVE: / EXPLANATION: / (conditionally) CRITIQUE: / optional COMMENT: lines, and the endpoint parses each into the field above. Because parsing lives in the generic layer, every game gets the coaching fields for free — a game only has to word its prompt.

Adding a game — the adapter contract

Games live in api/games/. A GameAdapter (games/base.py) owns one game's rules; the endpoint never changes. Register the adapter in games/__init__.py and it becomes selectable via the game field. The one invariant every adapter honours: it validates legality and detects endings with code, but never chooses a move — apply_move only accepts or rejects the text the model produced.

MethodResponsibility
initial_state()Serialized starting position (used when the client omits state).
load(state, history)Build an opaque game-state object; raise GameStateError (→422) on a bad state or GameDependencyError (→501) on a missing library. Replay history here if the game's rules need it.
serialize(game_state)Serialize the state object back to the string the client stores.
is_over / statusDetect mandatory endings; report game_over/result/termination/turn (+ game-specific keys).
apply_move(game_state, text)Validate text for the side to move; if legal, MUTATE the state and return an AppliedMove(notation, extras); else None.
legal_examples / promptA few legal moves for the error message; the instruction shown to the LLM. The prompt must ask for a MOVE: line and an EXPLANATION: line, and — when handed the opponent's user_move — a CRITIQUE: line, so the coaching fields are populated.

Bundled games

chess (games/chess_game.py)

python-chess is used strictly for legality — validating SAN/UCI moves, listing legal replies, detecting endings. State is a FEN. The SAN history is replayed to restore the board's move stack when it reproduces the FEN, which is what makes the repetition rules real across the stateless boundary: a bare FEN has no memory, so "fivefold" could never fire ("occurred once" every request). Mandatory endings (checkmate, stalemate, insufficient material, 75-move, fivefold) end the game automatically, including on ARRIVAL (reported instead of a confusing "Illegal move" against an empty legal list); claimable draws (threefold, 50-move) surface as can_claim_draw for the client to offer. python-chess is imported lazily, so a missing install only 501s a chess request — other games still work.

tictactoe (games/tictactoe.py)

A small, dependency-free second game that proves the endpoint is genuinely game-agnostic. State is nine cells in reading order (X/O/.) plus the side to move, e.g. "......... X". Cells are numbered 1-9 in reading order (top-left = 1); a move is that number (a1..c3 coordinates also accepted). Wins are the eight lines; a full board with no line is a draw. Impossible positions (counts off by more than one, since X moves first) are rejected with 422.
