UI layout

  • Left: chat display + text input with up/down history.
  • Right: stacked widget — NetworkFaceWidget (default) · FluidFaceWidget · NeuralFaceWidget (iris). Only the visible face animates.
  • Top: workspace, face toggle, fullscreen.
  • Bottom: camera snap, push-to-talk, TTS toggle + stats.

Voice queues

  • audio_queue (asyncio.Queue) — text chunks to TTS.
  • playback_queue — audio bytes awaiting aplay.
  • audio_fetch_task() — POSTs each chunk to TTS, queues result.
  • audio_worker_task() — pipes audio bytes to aplay subprocess.

Camera

CameraPreviewDialog via OpenCV VideoCapture on /dev/video0 at 1920×1080 60 fps MJPG. Ctrl+Esc snaps a frame; the user can optionally upload + add a prompt.

Input shortcuts

  • Esc — toggle PTT recording.
  • Alt+Esc — toggle TTS.
  • Ctrl+Esc — snap camera.
  • Slash commands: /clear, /shutdown, /reboot, /exit.

Stats

Timer every 5 s reads battery from /sys/class/power_supply/*/capacity and shows time + battery + network icon.

Concurrency

qasync.QEventLoop wraps the QApplication so asyncio and Qt cooperate. Async tasks for audio_fetch_task(), audio_worker_task(), send_chat_request(). Signal/slot pattern for cross-thread UI updates.

face/ — avatar package (CPU-rendered, QPainter raster)

Three interchangeable faces share one public API (set_mood · pulse · feed_audio · startle · wake · set_state) and one mood system (moods.py: idle/listen/think/speak/error/sleep, palettes lerped over 0.45 s transitions). The default is NetworkFaceWidget; the ◈ toolbar chip cycles to FluidFaceWidget (smoke oracle) and NeuralFaceWidget (iris eye).

  • NetworkFaceWidget (2026-07-12 "tamed" rework) — sparse MoE diagram: 42 nodes / 72 edges (was 111/276), 4 experts, one expert per pulse. A wavefront takes ~1.8 s to cross; activations decay fast enough that a thought visibly passes instead of lighting the whole net. Expert hues are the dark-jewel family: deep violet, crimson, dark blue, dark green. No bloom pass, no offscreen buffer, no wavefront bands, no status readout.
  • Mood palettes follow the same family as the web UI dark theme: idle/listen indigo-blue, think deep purple, speak dark green, error crimson.

Performance model (CM4, software raster — the lessons)

  • All three faces exist in a QStackedWidget, but animation timers are gated by showEvent/hideEvent — a hidden face costs zero CPU. (Previously all three simulated at 30 fps forever: the client pinned a core at ~113%.)
  • Full-screen software ops are the budget: nebula composite, wireframe skeleton, and vignette+scanlines are all cached pixmaps, re-rendered only when their inputs actually change (nebula scroll ~3 s, wireframe 200 ms, vignette per-resize). Grain is the only per-frame overlay blit.
  • Per-node QRadialGradient halos were replaced by flat translucent discs; active edges get one Bézier stroke, not glow+core, and are skipped below the skeleton's own brightness.
  • Adaptive frame rate: 20 fps active moods, 15 fps idle, 10 fps sleep.
  • Measured on-device: client steady-state ~20% of one core idle (was 113%), think ~60% during generation.

External dependencies

  • PyQt6 + qasync + PyOpenGL.
  • httpx, OpenCV (cv2), markdown, subprocess (arecord/aplay).
