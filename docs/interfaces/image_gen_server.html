interface / externals / image_generation / img_gen_server.py

Stable Diffusion XL Lightning server. Runs on a Mac (MPS) or Jetson (CUDA) box.

Endpoints

Method · PathBehaviour
POST /generate · POST /v1/images/generationsBody: {"prompt", "negative_prompt", "steps", "width", "height", "size", "seed", "clip_skip"}. Requires X-Ghost-Key. Size clamped to a VRAM-safe pixel budget. Returns {"data": [{"b64_json": ...}]}.
GET /health · GET /readyUnauthenticated. /health reports {ok, ready, load_error}; /ready is 200 when the model is loaded, 503 while warming, 500 on load failure.
Auth (2026-07-15). The deployed node (the Jetson, SD1.5 / CyberRealistic — this page still describes the earlier Mac SDXL variant elsewhere) binds 0.0.0.0 on the LAN and a generation monopolises the GPU for ~30–60s, so /generate now requires the fleet X-Ghost-Key (the same key the agent's own API uses; resolved from GHOST_API_KEY env, else ~/Data/AI/.ghost_api_key, else the server refuses to start — an explicit GHOST_API_KEY='' knowingly disables it on a trusted network). Auth is checked before the readiness gates, so an unauthenticated caller can never probe warmup/GPU state. Monitoring endpoints (/health, /ready) stay open. The agent stamps the key only on its image_gen httpx pool (llama.cpp pools don't check keys). A post-restart CUDA-teardown OOM now retries with backoff instead of parking until a manual bounce. Covered by tests/test_imggen_node_auth.py.

Model & scheduler

  • StableDiffusionXLPipeline from juggernaut_xl_lightning.safetensors.
  • EulerDiscreteScheduler with trailing timestep — required for Lightning models.
  • torch.float16 for memory.

Lifespan

asynccontextmanager loads the pipeline on startup, enables CPU offload + VAE slicing, runs a warmup pass. Garbage-collects + clears MPS/CUDA cache between generations. Sets PYTORCH_MPS_HIGH_WATERMARK_RATIO=0.0 for Mac.

Dependencies

  • diffusers (StableDiffusionXLPipeline, EulerDiscreteScheduler)
  • torch (float16, mps.empty_cache)
  • PIL
