Audit & hardening — May 2026

A deep correctness/security pass across the codebase. Every fix below ships with unit tests under tests/; the full suite (4300+ tests) is green and the live API was smoke-tested end-to-end on port 8000.

28+ fixes 11 new test files full suite green live-verified
Scope. These are correctness and security fixes — no feature changes. Two items change an observable contract and are called out inline: project DELETE now returns a truly empty 204 body, and the PRM must be retrained (its training-time feature state changed to remove label leakage). Everything else preserves documented behaviour while fixing a latent bug.

Critical — security & data integrity

AreaWhat was wrong → fixModule
Trajectory redactionOnly top-level string tool-args were redacted; secrets nested in list/dict args and the extra/validator_signal metadata reached the on-disk SFT corpus verbatim. Now recurses through nested containers, redacts both metadata dicts, and scrubs values under sensitive key names (password, authorization, …).distill/redact.py
SQL injectiontimeout_ms (an LLM-supplied arg) was f-string-interpolated into SET statement_timeout on an autocommit connection — a crash + injection sink. Now int-coerced, clamped, and parameterized.tools/database
ReDoS / scrubber leaksThe markdown code-fence extractor had a catastrophic-backtracking regex (a ~3 KB unterminated fence hung a worker). De-ambiguated + length-capped. Also: the fully-wrapped CDATA strip used rfind (leaked an embedded ]]>); and .strip('`') corrupted valid non-Python code with edge backticks.utils/sanitizer.py
Sandbox escapeThe forget tool's containment check used str.startswith, so a sibling dir like …/sandbox_evil looked "inside" …/sandbox — and a symlink could delete out-of-sandbox files. Now path-component containment (is_relative_to) and refuses to delete through symlinks.tools/memory
DB connection safetyOne shared psycopg2 connection per URI used concurrently across to_thread workers (protocol corruption). Now serialized behind a per-URI lock, with a one-shot reconnect on a dropped idle connection.tools/database

Concurrency — races on shared state

AreaWhat was wrong → fixModule
Docker sandboxensure_running() had no lock; concurrent tool calls raced container creation (409 name conflict, double ~2-min provisioning). Now lock-guarded, and a 409 is treated as "another process won" → adopt the existing container.sandbox/docker
Project advancerTwo ticks read the same READY leaf and marked it IN_PROGRESS only after an await — double-running a step. The leaf is now claimed atomically before any await, under a per-project lock.core/project_advancer
Reflection looprun() added a trajectory to the shared dedup set only after awaiting, so a concurrent reflect_one() double-reflected it. Now claimed before the await.reflection/loop.py
Agent loopPer-domain metacog competence was recorded once post-loop with the last tool's name + the turn-wide failure flag (mis-attributed on multi-tool turns) → now recorded per result. Plus: the Context-Shield summarizer no longer clobbers the loop-level payload (which truncated the Perfect-It generation), and the episode-archive write is tracked + drained on shutdown.core/agent
