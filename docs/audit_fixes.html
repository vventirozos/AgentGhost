Audit & hardening — May 2026

A deep correctness/security pass across the codebase. Every fix below ships with unit tests under tests/; the full suite (4300+ tests) is green and the live API was smoke-tested end-to-end on port 8000.

28+ fixes 11 new test files full suite green live-verified
Scope. These are correctness and security fixes — no feature changes. Two items change an observable contract and are called out inline: project DELETE now returns a truly empty 204 body, and the PRM must be retrained (its training-time feature state changed to remove label leakage). Everything else preserves documented behaviour while fixing a latent bug.

Round 2 — full-codebase gap sweep (2026-05-30)

A 14-agent parallel audit of the whole codebase surfaced a further set of gaps; all were fixed in this round. The dominant theme was built-but-unwired features (advertised/tested but never reached at runtime) and SSRF / isolation sinks where one guard existed but sibling sinks were missed.

Correctness & leaks

  • Vector retrieval reinforcement bumped the wrong ids (hashed doc text instead of the Chroma id), so spaced-repetition decay was dead for ingested-document chunks — now captures + uses the real id.
  • Streaming connection leak in llm.py: a 4xx/5xx leaked the streamed response (aclose() didn't cover raise_for_status()), exhausting the httpx pool under repeated errors — now an outer try/finally.
  • Idempotency guard recorded the call hash before the metacog gate could abort dispatch, blocking a later legitimate re-issue — now recorded only after dispatch.
  • find … | head was passed as literal args (the sandbox has no shell) — now wrapped in sh -c.
  • Plus: execute validator NameError (undefined logger), an LRU-bounded context-summary cache, dead writes/vars removed, and misleading docstrings corrected (router pool-gating, self-consistency offline-only).

Verification

  • Unit tests — each round has targeted tests under tests/. Round 2 adds test_ssrf_and_validators.py and test_redact_hardening_journal.py plus extensions, with negative-case regressions (SSRF bypass, rm -rf bypass, journal corruption, failed-graduation must-not-burn-lesson).
  • Full suite — pytest tests/ passes (4400+ tests, 0 failures after both rounds).
  • Live smoke test — the agent was restarted on port 8000 and exercised end-to-end: health/version/models endpoints, the projects CRUD + empty-204 path, a conversational turn, and a tool-using (execute) turn — all green.
