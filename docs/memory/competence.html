memory / competence.py — CompetenceProfile

Per-domain capability prior. The Level-1 (cross-domain transfer) component of the metacognition uplift.

Why per-(domain, tool)?

Aggregating success rates by raw tool name fragments the signal: list_files and read_file share a domain but live in different cells. Aggregating by tool alone loses the cross-tool generalisation MCTS needs. The dual key gives both: every record updates the leaf, the per-domain roll-up, AND the global cell — so estimate(domain, tool) can fall through whichever level has data.

Domain taxonomy

shell · sql · fetch · code · fs · memory · vision · other. Common synonyms (postgres → sql, bash → shell, python → code) are canonicalised at write time. Unknown domains land in other.

Prior

Beta(1, 1) — uniform. After ~100 samples the prior contributes <2% of the posterior mass; that's only load-bearing during the cold-start window. The CompositeConfidence scorer additionally shrinks toward neutral when the cell has <5 observations.

Persistence

Single JSON under the memory directory with atomic-rename writes. The format is keyed by "domain|tool" so a future schema change can be detected at load time.
