memory / graph.py — GraphMemory

SQLite-persisted knowledge graph mirrored into NetworkX for spreading-activation retrieval. Temporal triplet versioning handles belief updates.

Storage

  • SQLite at memory_dir/knowledge_graph.db.
  • In-memory networkx.MultiDiGraph rebuilt at startup via initialize_graph().

Spreading-activation retrieval

  1. Word-to-node fuzzy match: exact → substring → difflib.get_close_matches. The candidate node list is now a cached snapshot (_nodes_snapshot, invalidated on any edge mutation) instead of list(nodes()) rebuilt per query word — the graph is the only uncapped tier, so that O(nodes) rebuild ran per-turn (2026-07-07). Fragment guard (2026-07-22): the substring tier only seeds a node that is a FRAGMENT of the query word (ai from aiohttp, user from username) when it is ≥ 4 chars AND ≥ 75% of the word's length (_seed_containment_ok); the fuzzy tier drops stoplist hubs. That direction re-seeded generic ego hubs from unrelated words and blew the ego graph into every turn's context — now gone. The opposite direction (word inside a longer node, germ → germany) narrows rather than widens and stays unrestricted.
  2. 3-hop BFS from each seed.
  3. Score paths by sum of edge weights.
  4. Format winners as (subject)-[PREDICATE]-> (object) -[P2]-> (target).

Functional vs. non-functional predicates

_FUNCTIONAL_PREDICATES are SINGLE-valued: a new object supersedes the old one (which gets a valid_until stamp) instead of accumulating a contradiction. Every entry MUST be one-to-one for any subject — a multi-valued predicate in the set silently expires real knowledge on every new extraction.

  • Biographical: WORKS_AT, LIVES_IN, DRIVES, MARRIED_TO, BORN_IN, STUDIES_AT, LOCATED_IN, EMPLOYED_BY, HAS_AGE, HAS_LOCATION.
  • Operational (agent-written, single-valued by construction — a process has one status / one pid at a time): HAS_STATUS, STATUS, HAS_PID.

Everything else — LIKES, KNOWS, the HAS_* family, OWNS, IS — is multi-valued and accumulates.

OWNS / IS removed from the functional set (2026-07-22). Both were wrongly listed here and are MULTI-valued (you own many things; X is many things), so every new extraction expired a real prior fact: 19 of 21 live expired rows were OWNS — the operator's ownership history was silently lost, since restored. HAS_PET / HAS_TASK / HAS_FEATURE / HAS_NAME (written with the generic subject project) and HAS_STATE / HAS_FEN (position logs) are likewise deliberately absent. Conversely the genuinely single-valued operational predicates HAS_STATUS / STATUS / HAS_PID (and HAS_AGE / HAS_LOCATION) were MISSING and had been accumulating mutually-exclusive contradictions — now added. Every expiry is instrumented: logger.warning records subject predicate old → new on every supersession (write-time and merge-time), so an expiry is never silent. Covered by tests/test_graph_dataloss_fixes.py.

Forgetting — decay for the only uncapped tier (2026-07-07, #27c)

Unlike vector (5000-cap), episodes (500) and skills (50), the graph had no bound: every non-functional predicate accumulated forever, and weight-1 edges (one-off extractor noise) diluted retrieval as the graph grew. prune_stale_edges(max_age_days=45, keep_min_weight=1) deletes currently-valid edges whose weight ≤ keep_min_weight AND whose timestamp is older than the age threshold — from both SQLite and the mirror. Reinforced edges (weight > threshold) survive regardless of age: weight IS the decay signal, previously stored but never used for forgetting. It runs from the dream cycle (best-effort, never fails a dream) and reports the count in the Dream-Complete line. Covered by tests/test_graph_forgetting.py. (A persistent token→node inverted index — replacing the substring/difflib scan entirely — remains a further optimization.)

Concurrency

threading.RLock guards both SQLite and the networkx mirror. initialize_graph() rebuilds the entire mirror from disk.
