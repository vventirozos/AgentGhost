memory / contradiction_log.py — ContradictionLog

Append-only audit log of belief revisions. Drives "I previously thought X, but updated because…" explanations.

Storage

  • JSON at memory_dir/contradiction_log.json.
  • FIFO cap MAX_ENTRIES = 200.
  • Atomic temp + os.replace; threading.RLock.

Load hardening (2026-07-22)

_load() distinguishes three outcomes rather than the old except Exception: return [] that made an unreadable file behave exactly like an absent one — so the next record() atomically overwrote 200 entries of belief-revision history with a single entry, silently:

  • absent → [] (normal cold start);
  • corrupt (bad JSON / wrong type / undecodable bytes) → move the raw bytes to a timestamped contradiction_log.corrupt-*.json sidecar, WARNING, start clean;
  • present-but-unreadable (OSError — EIO / EACCES) → return [] for reads but set _degraded (logged at ERROR); while set, _save() REFUSES to write. The flag clears automatically as soon as a read succeeds.

__init__ now probes via _load() at construction — not lazily on the first record() — specifically because clear() writes without reading: a lazy check left one path that could overwrite a corrupt / unreadable file without preserving it. Tests: tests/test_memory_store_durability.py.

Entry shape

{
  "timestamp":   "ISO",
  "new_fact":    str,
  "superseded":  [{ "id": str, "text": str }],
  "deleted_ids": [str],
  "reason":      str
}
