Retention

MAX_EPISODES = 500. When exceeded the oldest non-lesson, non-consolidated episodes are deleted first; a fallback then deletes the oldest of any kind so the cap is always enforced. MAX_ACTIONS_PER_EPISODE = 20.

Action truncation keeps HEAD+TAIL (2026-07-22): when an episode exceeds MAX_ACTIONS_PER_EPISODE the writer (_truncate_actions) keeps the first TRUNCATION_HEAD_ACTIONS = 5 actions, inserts one (actions elided) marker row recording how many were dropped, and keeps the rest of the budget from the tail. The old actions[:20] dropped the tail where the recovery lives ("tried X, failed, switched to W") while the episode row still asserted outcome_success = 1 — so the stored record showed only the failed exploration (16 of 139 live episodes sat at exactly 20 actions). Logs a warning on every truncation; the marker row is stored with success = 1 so it never reads as a failed action to the recovery signal.

Orphan reaping (2026-06-11): episode_actions has a declared but un-enforced FK (no cascade), so deleting an episode leaves its action rows orphaned. record_episode now reaps orphans unconditionally on every write via an indexed anti-join (idx_ea_episode) — not only when an insert breaches the cap, as before — so orphans from non-cap delete paths (consolidation, external deletes) can no longer accumulate unboundedly. Covered by tests/test_memory_growth_bounds.py.

Semantic recall wiring (2026-07-07)

The write path ingests every episode's trigger into the vector store expressly so this tier can recall episodes by meaning. The MemoryBus._fetch_episodic read path now passes its vector store to search_similar, activating _vector_search (relevance-gated, best-distance-per-episode). Previously the bus called search_similar with no vector_memory, so recall always used the substring fallback over only the 100 most-recent episodes — episodes with no keyword overlap to the query (nearly half the live collection) were unreachable. search_recoveries (System-3 crisis pivots) inherits the same fix wherever it is called with the store.

Memory-substrate review (2026-07-22)

Making episodic recall actually reachable and comparable across paths (the HEAD+TAIL action-truncation fix is noted under Retention above). Covered by tests/test_episodes_recall_fixes.py.

  • search_recoveries was structurally dead. It required a non-empty lesson, but the only production writer (core/agent.py::_record_episode_safe) never sets one — 0 of 145 live episodes qualified, so every crisis pivot silently got []. It now derives the recovery signal from fields the live schema DOES carry (_recovery_evidence): chiefly "ended successfully yet contains a FAILED tool call" — the literal "tried X, it broke, switched to W" signature, matching 38 of 145 live episodes — plus trouble-narrating outcome text (error / fixed / retried / fell back …) for episodes whose action rows are absent. Each hit is tagged recovery_evidence; strong evidence ranks first, and only if nothing qualifies does it fall back to plain relevant successes, so a weak answer beats the previous guaranteed-empty one.
  • Vector twins now store an ISO-8601 timestamp. _ingest_episode_vector stamps each episode's vector-store metadata with get_utc_timestamp(), like every other writer into the store. A float epoch here rendered in the prompt as [1783488280.73] (VectorMemory._render_item prints the raw value) next to ISO-8601 neighbours — unreadable as a date for exactly the tier where recency matters most. (The SQLite episodes.timestamp column stays REAL; this is the vector metadata only.)
  • A failed vector ingest now logs at warning (was debug). An embedding failure left an episode with a valid row id but permanently invisible to semantic recall, with no signal on the live stream (9 live episodes in that state). The new reconcile_vector_index helper anti-joins the indexed episode_ids against the newest rows and re-ingests the missing twins.
  • Comparable relevance scoring on both paths. Recall now stamps a float relevance_score in [0, 1] on BOTH paths: the vector path uses 1 − distance (floored at MIN_VECTOR_RELEVANCE = 0.2, so semantically unrelated hits are dropped rather than injected as flat 1.0 matches), the substring path uses the fraction of query words matched — previously a raw integer count, so a caller ranking across paths compared 3 against 0.9. A mild recency multiplier (_shape_by_recency, half-life 14 d, RECENCY_WEIGHT = 0.15) breaks ties without overriding match quality, and format_episode prefixes a relative age ([3d ago]) so the model can tell "how the system behaves" from "how it used to". The substring fallback's scan depth rose from a hardcoded 100 to FALLBACK_SCAN_LIMIT = MAX_EPISODES: with a 500-cap store, everything older than the newest 100 episodes had been unreachable by that path — and unreachable full stop when its vector twin was missing.
