Tooling & scripts

Claude trainer

scripts/claude_trainer.py drives the agent's interactive learning loop using Claude as the teacher: it poses tasks, grades the answers, and issues corrections engineered to fire the agent's own classify_user_correction predicate — then re-tests in a fresh conversation to measure whether the lesson stuck.

teacher = Claude student = Ghost Agent closes the chat-learning loop no weight updates
What it trains, and what it does not. The agent's interactive self-improvement path is: a user message classified as a correction promotes the prior assistant turn to FAILED (agent.py:_maybe_promote_prior_turn_via_user_correction), which schedules a fire-and-forget reflection that distils a lesson into SkillMemory (skills_playbook.json). A later turn retrieves that lesson through the RRF memory bus. The trainer exercises exactly this loop. It teaches retrieval-backed lessons, not model weights — there is no fine-tuning.

The critical detail: making a correction actually trigger learning

The agent only learns from a correction when distill/user_correction.py::classify_user_correction returns is_correction=True, which requires both signals:

SignalRequirement
A — phraseAn anchored correction phrase ("No, that's not right…") at the very start of the message.
B — rephraseJaccard token overlap ≥ 0.40 between the prior user question and the correction (the correction must restate the ask).

A naive "you're wrong, the answer is X" satisfies A but misses B, so it silently fails to record any lesson. build_correction_message() therefore restates the question and is trigger-first: it appends the correct answer only when the extra tokens don't dilute the overlap below 0.40; for short questions it degrades to opener+question (learning still fires — the answer simply isn't embedded in the trigger turn). When the package is importable the trainer verifies each correction against the real classifier before sending via correction_will_trigger().

Per-task flow

1. ASK        send the question to the live agent (/api/chat)
2. JUDGE      Claude grades the answer vs. an authoritative reference (0.0–1.0)
3. CORRECT    if wrong, send a both-signals correction → agent promotes the
              prior turn to FAILED and schedules a reflection that writes a lesson
4. RE-JUDGE   grade the in-context follow-up answer
5. WAIT       pause so the fire-and-forget reflection lands
6. RETEST     re-ask in a FRESH conversation; Claude scores retention
Teacher = official Anthropic SDK, in an isolated venv. The trainer calls Claude through the official anthropic SDK (messages.parse() with Pydantic schemas for structured grading/curriculum output, plus adaptive thinking, on claude-opus-4-8) — not raw HTTP. The repo's shared .agent.venv pins anthropic 0.4.1 (pre-Messages-API, required by evalplus/tau_bench), so the trainer runs from a dedicated .trainer.venv to avoid disturbing that pin. The Ghost client still uses httpx directly — it speaks the local agent's OpenAI/Ollama API, which is not the Anthropic API.

Usage

# One-time: isolated venv with a modern anthropic SDK
python -m venv .trainer.venv
.trainer.venv/bin/pip install anthropic pydantic 'httpx[socks]'

# Claude invents a 6-task curriculum on a topic, then trains:
ANTHROPIC_API_KEY=sk-... \
.trainer.venv/bin/python scripts/claude_trainer.py --topic "PostgreSQL internals" --tasks 6

# Train from a curriculum file (JSON list of {question, reference, domain}):
.trainer.venv/bin/python scripts/claude_trainer.py --curriculum mytasks.json

# Faster: skip the fresh-conversation retention re-test:
.trainer.venv/bin/python scripts/claude_trainer.py --topic "Python asyncio" --no-retention

# Write a machine-readable report:
.trainer.venv/bin/python scripts/claude_trainer.py --topic "TCP/IP" --report /tmp/train.json

The report

The console summary (and the optional --report JSON) tracks: first_try_correct, needed_teaching, corrections_that_triggered_learning (the count whose corrections actually fired the predicate — watch this; if it is below needed_teaching, some lessons were never recorded), retention_tested, and retention_improved (fresh re-asks that scored materially higher than the first attempt — the real evidence a lesson stuck).

Caveats. Requires the live agent (see Install & Run) and an ANTHROPIC_API_KEY. Reflection is judge-gated and fire-and-forget, so a lesson can land slightly after the correction turn — that's why retention is measured after a delay in a fresh conversation. Bad corrections poison retrieval, so teacher-grading quality matters; the curriculum prompt asks Claude for single-answer, verifiable questions.
